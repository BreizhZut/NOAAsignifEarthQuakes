Introduction

This package is the result of a project centered around a dataset obtained from the U.S. National Oceanographic and Atmospheric Administration (NOAA). This dataset is focussed on significant earthquakes around the world and contains information about 5,933 earthquakes over an approximately 4,000 year time span.

The objective of this package is to:

  1. Efficently process the data
  2. Provide easy to use function for dedicated explorattory analysis

Data source

National Geophysical Data Center / World Data Service (NGDC/WDS): Significant Earthquake Database. National Geophysical Data Center, NOAA. (doi:10.7289/V5TD9V7K)

Reading and cleaning the data

We use 2 main function to read and clean the data: 1. load_NOAA_db : read the raw NOAA file 1. eq_clean_data: process the resulting data frame using support finctions: * eq_build_date: to create the date feature from input year, month and day * eq_build_locaton: to proccess the location feature in a more human readable way

Reading the data: load_NOAA_db

We odtain a table with 47 features and 6066 observations. We describe the schema of the raw data in detail in the appendix.

Cleaning the data; eq_clean_data

Mainly this funcions consist of processing existing the features with the helper function above and select features reducing the raw data to 7 features but still 6066 observations:

  1. DATE : date of the earthqukes event
  2. COUNTRY : country where the earthquake occured
  3. LOCATION_NAME : location of the earthquake
  4. LONGITUDE : Longitude coordinate of the Earthquake epicenter
  5. LATITUDE : Latitude coordinate of the Earthquake epicenter
  6. DEATHS : Total number of fatalities caused by the earthquakes
  7. MAG : Equivelent Richter scale Magnitude
noaa_clean <- eq_clean_data(noaa_raw)

Building a Timeline

Filtering the data with timeline_data

This function consist on preparing the data prioir to building the timeline. It enable to:

  1. Drops features unnecessary to the timeline (e.g. LATITUDE and LONGITUDE)
  2. filter within a date range specified with keyword dmin and dmax
  3. filter 1 or several countries with keyword countries
  4. Additionnaly it create a feature MAG_RANK giving the rank order of earthquake by country in descreasing order of Magnitude
filt_noaa <- noaa_clean %>%
  timeline_data(dmin='2010-01-01',dmax='2011-01-01',countries=c("USA","China"))

Showing timelines geom_timeline

For 1 country without easthetics

g_usa <- geom_timeline(noaa_clean,
                       countries='USA',
                       xmin='2000-01-01',
                       xmax='2017-01-01')

For multiple countries with easthetics

This geom takes for optional aesthetics

  • y: a factor variable such as COUNTRY to display mutiple timelines
  • size: a continuous variable scaling the point symbol (usually set to MAG)
  • fill: a continuous variable scaling the color symbol (usually set to DEATHS)
g_usachina <- geom_timeline(noaa_clean,
              mapping = ggplot2::aes(
                y=COUNTRY,
                fill=DEATHS,
                size= MAG
              ),
              countries=c('USA',"China"),
              xmin='2000-01-01',
              xmax='2017-01-01')

Building a Map

Setting a use case

To create a map we have to filter the processed data for a country and specific date range so as to avoid overflowing the visualisation.

filt_noaa <- noaa_clean %>% dplyr::filter(COUNTRY == "MEXICO" & lubridate::year(DATE) >= 2000)

Map with Date popup eq_map

Each point correspond to an earthquake with the size of the circle represnting the magnitude.

eq_map(filt_noaa,annot_col = "DATE")

Map with annotated popup eq_map with eq_create_label

Summary

The NOAAsignifEarthQuakes performs in a quite straitforward way 3 things

  • Read and prepare the raw NOAA data
  • Enable the user to expore this date using a efficient timeline visualization
  • Enable the user to explore the event location

Appendix: Raw data codebook

We give a short description of the feature present in the raw data. We indicate the column type as we set it while reading the data.

  1. Earthquake Id and type
    • I_D : [Character] unique id for the earthquake
    • FLAG_TSUNAMI : [Character] Categorical set to Tsu if the earthquake generated a Tsunami (set to NA otherwise)
  2. Date Time variable
    • YEAR : [Integer] 4 digit year corresponding the event, range -2150 to 2018
    • MONTH : [Integer] 2 digit month corresponding the event, could be NA
    • DAY : [Integer] 2 digit day of the month corresponding to the event
    • HOUR : [Integer] 2 digit hour corresponding the event
    • MINUTE : [Integer] 2 digit minute corresponding to the event
    • SECOND : [Numeric] 2 digit seconds corresponding to the event
  3. Earthquake properties
    • FOCAL_DEPTH : [Integer] Depth of the earthquake [km] (Valid values: 0 to 700 km)
    • EQ_PRIMARY : [Numeric] Equivalent Magnitude, take the value from one of the column below (supposidly only up to one contains a non-missing value)
    • EQ_MAG_MW : [Numeric] Magnitude based on the moments magnitue scale (Valid values: 0.0 to 9.9)
    • EQ_MAG_MS : [Numeric] Surface-wave magnitude (Valid values: 0.0 to 9.9)
    • EQ_MAG_MB : [Numeric] Compressional body wave (P-wave) magnitude (Valid values: 0.0 to 9.9)
    • EQ_MAG_ML : [Numeric] Standard Richter Magintude (Valid values: 0.0 to 9.9)
    • EQ_MAG_MFA : [Numeric] Magnitude estimated for field area, before sesmic instruments were in use (Valid values: 0.0 to 9.9)
    • EQ_MAG_UNK : [Numeric] Magnitude estimated using and unidentified method (Valid values: 0.0 to 9.9)
    • INTENSITY : [Character] Modified Mercalli Intensity (Valid values: 1 to 12)
  4. Location
    • COUNTRY : [Character] Country name where the event occured
    • STATE : [Character] State where the event occured,
    • LOCATION_NAME : [Character] Location name ex: City or geographical landmark
    • LATITUDE : [Double] Latitude coordinates of the event
    • LONGITUDE : [Double] Longitude coordinates of the event
    • REGION_CODE : [Character] Event location, contains country name and locality
  5. Death and injuries
    • DEATHS : [Integer] Number of deaths
    • DEATHS_DESCRIPTION : [Character] Factorial code 1 to 4
    • MISSING : [Integer] Number of Missing persons
    • MISSING_DESCRIPTION : [Character] Factorial code 1 to 4
    • INJURIES : [Integer] Number of injured persons
    • INJURIES_DESCRIPTION : [Character] Factorial code 1 to 4
  6. Property Damage
    • DAMAGE_MILLIONS_DOLLARS : [Numeric] Damage amount in M$
    • DAMAGE_DESCRIPTION : [Character] Factorial code 1 to 4
    • HOUSES_DESTROYED : [Integer] Number of Houses destroyed
    • HOUSES_DESTROYED_DESCRIPTION : [Character] Factorial code 1 to 4
    • HOUSES_DAMAGED : [Integer] Number of Houses damaged
    • HOUSES_DAMAGED_DESCRIPTION : [Character] Factorial code 1 to 4
  7. Total Death and injuries including related events (Tsunami, Volcano etc..)
    • TOTAL_DEATHS : [Integer] Number of deaths
    • TOTAL_DEATHS_DESCRIPTION : [Character] Factorial code 1 to 4
    • TOTAL_MISSING : [Integer] Number of Missing persons
    • TOTAL_MISSING_DESCRIPTION : [Character] Factorial code 1 to 4
    • TOTAL_INJURIES : [Integer] Number of injured
    • TOTAL_INJURIES_DESCRIPTION : [Character] Factorial code 1 to 4
  8. Total Property Damage including related events (Tsunami, Volcano etc..)
    • TOTAL_DAMAGE_MILLIONS_DOLLARS : [Numeric] Damage amount in M$
    • TOTAL_DAMAGE_DESCRIPTION : [Character] Factorial code 1 to 4
    • TOTAL_HOUSES_DESTROYED : [Integer] Number of Houses destroyed
    • TOTAL_HOUSES_DESTROYED_DESCRIPTION : [Character] Factorial code 1 to 4
    • TOTAL_HOUSES_DAMAGED : [Integer] Number of Houses damaged
    • TOTAL_HOUSES_DAMAGED_DESCRIPTION : [Character] Factorial code 1 to 4
